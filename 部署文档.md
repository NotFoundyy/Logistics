# 快递物流管理系统部署文档

本文档按当前项目结构编写，按步骤执行即可完成本地运行和基础部署。

## 1. 项目结构

- 后端目录：项目根目录（`pom.xml` 所在目录）
- 前端目录：`web`

## 2. 环境要求

- JDK：17
- Maven：使用项目自带 `mvnw` / `mvnw.cmd`
- Node.js：18 及以上（建议 20）
- MySQL：8.x
- 操作系统：Windows / Linux

## 3. 首次准备

### 3.1 创建数据库

先启动 MySQL，然后执行：

```sql
CREATE DATABASE IF NOT EXISTS logistics
DEFAULT CHARACTER SET utf8mb4
COLLATE utf8mb4_general_ci;
```

### 3.2 配置后端

编辑 `src/main/resources/application.properties`，至少确认以下配置正确：

- `spring.datasource.url`
- `spring.datasource.username`
- `spring.datasource.password`

说明：

- 当前项目通过 `schema.sql` + `data.sql` 初始化数据。
- 如果你是新库，保持初始化开启即可。

### 3.3 配置前端（高德地图）

编辑 `web/.env.local`：

```env
VITE_AMAP_KEY=你的高德Key
VITE_AMAP_SECURITY_JS_CODE=你的高德安全密钥
```

## 4. 本地开发启动

建议开两个终端。

### 4.1 启动后端

在项目根目录执行：

```powershell
$env:JAVA_HOME='E:\JDK17'
$env:Path="$env:JAVA_HOME\bin;$env:Path"
.\mvnw spring-boot:run
```

启动成功后：

- 后端地址：`http://localhost:8080`
- Swagger：`http://localhost:8080/swagger-ui/index.html`

### 4.2 启动前端

在项目根目录执行：

```powershell
cd web
npm install
npm run dev
```

启动成功后访问：`http://localhost:5173`

## 5. 生产打包与部署

### 5.1 打包后端

项目根目录执行：

```powershell
.\mvnw clean package -DskipTests
```

产物：`target/Logistics-0.0.1-SNAPSHOT.jar`

运行：

```powershell
java -jar target/Logistics-0.0.1-SNAPSHOT.jar --server.port=8080
```

### 5.2 打包前端

```powershell
cd web
npm install
npm run build
```

产物目录：`web/dist`

### 5.3 Nginx 反向代理示例

将 `web/dist` 部署到 Nginx 静态目录，示例配置：

```nginx
server {
    listen 80;
    server_name localhost;

    root /data/logistics/web/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 6. 常见问题

### 6.1 `UnsupportedClassVersionError`

原因：JDK 版本不一致。

处理：确认 IDEA、Maven、运行时都使用 JDK17。

### 6.2 `Failed to configure a DataSource`

原因：数据库连接配置缺失或错误。

处理：检查 `application.properties` 数据源配置和数据库服务状态。

### 6.3 `Port 8080 was already in use`

处理：

- 关闭占用 8080 的进程；或
- 修改 `server.port`。

### 6.4 `npm ERR! enoent package.json`

原因：当前目录不在前端目录。

处理：先 `cd web` 再执行 `npm run dev`。
